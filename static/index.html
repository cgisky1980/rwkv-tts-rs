  "success": false,

  "message": "详细错误信息"

💡 使用示例

cURL 示例：

# 基本请求
curl -X POST http://localhost:8080/api/tts \
  -H "Content-Type: application/json" \
  -d '{
    "text": "你好，这是一个测试文本",
    "temperature": 1.0,
    "top_p": 0.3,
    "speed": 1.0
  }'

# Zero-shot模式（带文件上传）
curl -X POST http://localhost:8080/api/tts \
  -F "text=你好，这是一个测试文本" \
  -F "zero_shot=true" \
  -F "prompt_text=这是一个温和的女性声音" \
  -F "refAudioFile=@reference.wav"

Python 示例：

import requests
import base64

# 基本请求
response = requests.post('http://localhost:8080/api/tts', json={
    'text': '你好，这是一个测试文本',
    'temperature': 1.0,
    'top_p': 0.3,
    'speed': 1.0,
    'age': 'youth-adult',
    'gender': 'female',
    'emotion': 'HAPPY'
})

if response.status_code == 200:
    result = response.json()
    if result['success']:
        # 保存音频文件
        audio_data = base64.b64decode(result['audio_base64'])
        with open('output.wav', 'wb') as f:
            f.write(audio_data)
        print(f"音频生成成功，耗时: {result['duration_ms']}ms")
    else:
        print(f"生成失败: {result['error']}")

📝 注意事项

  • 文本长度建议不超过1000个字符，过长的文本可能导致生成时间过长
  • Zero-shot模式需要上传参考音频文件，支持WAV、MP3、FLAC、M4A格式
  • 参考音频文件大小建议不超过10MB
  • 相同的seed值可以复现相同的生成结果
  • temperature值越高，生成的语音越随机；越低，越稳定
  • API响应中的audio_base64字段包含WAV格式的音频数据
  • RTF (Real Time Factor) 表示生成速度，值越小表示生成越快
